Task 3

User & Group Management

Groups & access (RBAC)

GroupTypical access
StudentsLMS, student email, lab PCs, limited storage
LecturersCourse mgmt, databases (read), shared drives
Admin-StaffAdmin systems, finance/HR shares, email
IT-AdminsFull server & management-tool access
Library/Finance/HRDepartment-specific apps & shares only

Least privilege

  • Permissions to groups, not people
  • Delete rights restricted to a few accounts
  • LMS roles: student / teacher / manager

Secure lifecycle

  • Central AD / LDAP with SSO + MFA
  • Joiner–mover–leaver provisioning
  • Audit logging of every change

Directly prevents accidental deletion of student records: limited delete rights + least privilege + full audit trail.

Task 5

Security Management

Identity & access

  • 12+ char passwords, lockout, 2FA
  • Password manager + SSO
  • RBAC, least privilege, VPN for remote

Network & endpoint

  • Next-gen firewall, default-deny
  • VLANs: students/staff/servers/labs/DMZ
  • Central antivirus · TLS · WPA3 Wi-Fi

People & logs

  • Security awareness training
  • Phishing simulations
  • Central logs, reviewed regularly

Maintaining the CIA triad

CConfidentiality

  • Encryption (TLS, disk), RBAC, 2FA, VPN

IIntegrity

  • Checksums, version control, change mgmt, audit logs

AAvailability

  • Redundancy, load balancing, backups, DR, DDoS protection

Task 6 & Task 7

Automation & User Support

Automation Management

  • Account provisioning: PowerShell + AD (CSV bulk-create)
  • Backup scheduling: Task Scheduler / cron (nightly 12 AM)
  • Monitoring & alerts: Zabbix (e.g. CPU > 90%)
  • Patching: WSUS / Intune
  • Log rotation, cert renewal, report generation

Saves time, reduces human error, frees the small IT team for higher-value work.

User Support & Education

  • osTicket help desk: portal, email, phone, walk-in
  • Two-tier support: L1 common · L2 complex / escalation
  • SLAs with response / resolution targets by priority
  • Induction + cybersecurity awareness programmes
  • Self-service knowledge base: FAQs & user guides

Resolves limited technical support; every request logged & tracked.

Conclusion

How We Solve ABC Institute's Problems

Problem identifiedRecommended solution
Accidental deletion of recordsLeast-privilege RBAC, restricted delete rights, audit logs, soft-delete + tested backups
Weak password practicesEnforced password policy, 2FA, SSO, password manager
Website downtime at registrationLoad balancing / redundancy, caching + CDN, monitoring, capacity planning
Lack of backup proceduresAutomated 3-2-1 strategy, off-site/cloud + immutable copies, verified restores
Limited technical supportTicketing system, tiered help desk with SLAs, documentation & training
No disaster recovery planDocumented DR plan with RTO/RPO targets, off-site failover, annual DR drills

Roadmap & priorities

Now: 2FA, automated backups & help desk  →  Mid-term: monitoring & scaling infrastructure  →  Long-term: disaster-recovery investment. A secure, scalable IT environment that grows with the institute.
